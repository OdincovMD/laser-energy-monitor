# Первичная проверка Ophir / StarLab log

Цель проверки - подтвердить, что StarLab пишет файл `Data_log.txt`, а `Laser Energy Monitor` читает из него новые значения.

## Перед запуском

1. Подключите датчик Ophir к официальной программе StarLab.
2. Включите в StarLab логирование результатов.
3. Убедитесь, что файл `Data_log.txt` создается и пополняется строками.
4. Запомните путь к файлу, обычно это:

```text
C:\Users\<user>\Documents\StarLab\Data_log.txt
```

## Проверка в приложении

1. Запустите `LaserEnergyMonitor.Wpf.exe`.
2. В блоке Beam выберите `BeamGage SDK`.
3. Нажмите `Scan`, выберите нужный BeamGage source и нажмите `Connect`.
4. В блоке Ophir выберите `StarLab Log File`.
5. Нажмите кнопку выбора файла и укажите `Data_log.txt`.
6. Нажмите `Self-Test`.
7. Проверьте, что карточка Ophir сообщает, что StarLab log file готов или читается.
8. Запустите короткую измерительную сессию и проверьте, что значение Ophir меняется в UI и попадает в Excel.

## Что сохранить при проблеме

- `application.log`
- итоговый `.xlsx`
- `measurement-session.RawData.csv`
- скриншот StarLab с включенным логированием
- первые строки и последние строки `Data_log.txt`

## Типовые причины проблем

- В StarLab не включена запись лога.
- Выбран не тот файл.
- В файле еще нет заголовка таблицы.
- В конфигурации указана колонка энергии, которой нет в файле.
- StarLab пишет с задержкой flush, поэтому новые строки появляются пачками раз в несколько секунд.
