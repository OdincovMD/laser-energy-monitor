# Текущий статус реализации

## Что реально есть в репозитории

- решение `LaserEnergyMonitor.sln`;
- WPF-приложение `LaserEnergyMonitor.Wpf` под `.NET Framework 4.8`;
- фиксированная целевая платформа `x86`;
- доменная модель измерений, синхронизации и стационарности;
- слой оркестрации `LaserEnergyMonitor.Application`;
- интеграции `BeamGage` и `Ophir` с предварительной проверкой runtime и smoke-test сценариями;
- replay-режим и симуляция источников для безопасной отладки без оборудования;
- экспорт результатов в `.xlsx` через `Open XML`, при этом данные сначала пишутся во вспомогательные CSV-файлы;
- набор unit-тестов для доменной логики и слоя приложения.

## Что подтверждено локально

- решение собирается в `Release|x86` без ошибок и предупреждений;
- unit-тесты проходят: `8/8`;
- для `Ophir` подтверждено, что COM ProgID `OphirLMMeasurement.CoLMMeasurement` может быть найден и активирован на текущей машине;
- при этом наличие runtime не заменяет живую проверку подключенного оборудования.

## Что уже не является заглушкой

- предварительная проверка runtime и smoke-test для `BeamGage`;
- предварительная проверка runtime и smoke-test для `Ophir`;
- WPF-интерфейс как основная точка входа приложения;
- построение итогового `.xlsx` файла на базе вспомогательных CSV-файлов.

## Что остается риском до проверки на стенде

- видимость реального `BeamGage` на целевой машине;
- видимость реального `Ophir / Pulsar-4` по USB и корректность выбора канала;
- отсутствие конфликта с программным обеспечением вендора, которое может удерживать устройство;
- согласованность отметок времени от вендора и качество потока реальных выборок;
- поведение длинной сессии на реальном оборудовании.

## Практический следующий шаг

1. Собрать `Release|x86`.
2. Запустить приложение на целевой машине.
3. Выполнить `Self-Test`.
4. Выполнить `BeamGage Test`, если используется реальный `BeamGage`.
5. Выполнить `Ophir Smoke-Test`, если используется реальный `Ophir / Pulsar-4`.
6. Провести короткую реальную сессию и проверить артефакты в папке `output`.
