# Текущий статус реализации

## Что уже сделано

- создана solution `LaserEnergyMonitor.sln`;
- заведены проекты по слоям:
  - `Wpf`
  - `Application`
  - `Domain`
  - `Infrastructure`
  - `Infrastructure.BeamGage`
  - `Infrastructure.Ophir`
  - `Infrastructure.Excel`
  - `Infrastructure.Logging`
- зафиксирован target:
  - `.NET Framework 4.8`
  - `WPF`
  - `x86`
- реализовано доменное ядро:
  - модель одиночного измерения;
  - модель синхронной пары;
  - модель событий серии;
  - настройки сессии;
  - кольцевое окно `RollingWindow`;
  - синхронизатор `TimeWindowMeasurementSynchronizer`;
  - детектор стационарности `RollingStationarityDetector`;
- реализован orchestration-слой:
  - `MeasurementSessionService`;
  - lifecycle и события серии;
  - реакция на рассинхрон и fault;
- реализован минимальный прототип UI:
  - старт и стоп;
  - параметры `N`, порогов и окна синхронизации;
  - вывод текущих значений и средних;
  - журнал событий;
- реализован прототипный exporter:
  - вместо финального `XLSX` пока пишет три CSV-файла:
    - `*.RawData.csv`
    - `*.Events.csv`
    - `*.Summary.csv`
- добавлены симулированные источники измерений для безопасной отладки pipeline без оборудования.

## Что пока сделано как заглушка

- реальная интеграция с `Beam Gage`;
- реальная интеграция с `Ophir / Pulsar-4`;
- финальный `XLSX` exporter на `Open XML SDK`.

Эти части уже имеют выделенные проекты и точки расширения, но еще не подключены к vendor SDK.

## Почему так

Это сделано специально, чтобы:

- сначала стабилизировать архитектуру;
- проверить поток данных, синхронизацию и логику стационарности;
- не смешивать разработку ядра с нестабильной интеграцией драйверов;
- перейти к hardware integration отдельным контролируемым этапом.

## Следующий практический этап

Следующий рекомендуемый шаг:

1. сделать integration spike для `Beam Gage`;
2. сделать integration spike для `Ophir / Pulsar-4`;
3. подтвердить способ получения timestamp / sequence;
4. заменить симулированные источники реальными адаптерами;
5. заменить временный CSV-shadow exporter на настоящий `XLSX`.
