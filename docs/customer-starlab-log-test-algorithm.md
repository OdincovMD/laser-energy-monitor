# Алгоритм проверки Ophir через StarLab log у заказчика

## Подготовка

1. Подключить датчик Ophir и открыть StarLab.
2. Убедиться, что StarLab видит датчик и показывает измерения.
3. Включить логирование результатов.
4. Проверить, что файл `Data_log.txt` обновляется.

## Проверка в Laser Energy Monitor

1. Запустить приложение.
2. Для Beam выбрать `BeamGage SDK`, выполнить `Scan`, выбрать источник и нажать `Connect`.
3. Для Ophir выбрать `StarLab Log File`.
4. Указать файл `Data_log.txt`.
5. Нажать `Self-Test`.
6. Запустить короткую сессию измерения.
7. Открыть итоговый Excel и проверить, что в колонках Ophir значения меняются.

## Критерий успеха

- StarLab продолжает писать лог.
- Приложение читает новые строки без перезапуска.
- В UI видно актуальное значение Ophir.
- В Excel записаны разные значения Ophir, а не одна повторяющаяся константа.

## Что вернуть разработчику при проблеме

- `application.log`
- итоговый `.xlsx`
- `measurement-session.RawData.csv`
- путь к выбранному `Data_log.txt`
- небольшой фрагмент начала и конца `Data_log.txt`
